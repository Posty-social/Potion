# Potion Prompt Files

These files preserve the working prompts and product decisions that shaped the
current build. They are intentionally written as implementation prompts so the
next person can continue the project without rereading the whole chat.

- `product-brief.md` - what Potion is and who it is for.
- `architecture.md` - data, auth, routing, realtime, files, MCP, and search.
- `deployment-variables.md` - variables/secrets David's boss needs to supply.
- `privacy-reference.md` - how to treat imported private chat content.
